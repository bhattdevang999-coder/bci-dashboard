Atlas — Novelle Operations Onboarding

For: the agency team running Novelle's Amazon operations · From: Devang (Novelle) · Date: May 2026

Why you're reading this

You're going to run Novelle's Amazon operations — listings, catalog hygiene, PPC analysis, weekly reporting. You can do that work in Helium 10, in Excel, in Seller Central directly, and on your laptop's notes app. That's how every brand you've worked with does it.

Novelle is asking you to do most of that same work inside a platform we've built called Atlas. This doc explains what Atlas is, what changes for you, and — honestly — what's in it for you.

The short version:

  • For you: Atlas takes the grunt work off your plate (weekly reports auto-generate, catalog audits surface issues ranked by dollar impact, listings get drafted faster). You spend more time on judgment calls, less on assembling data.
  • For Novelle: every decision you make gets captured so the brand carries institutional memory you can build on. Future you, future agency hires, future Novelle hires — they can see what was tried, what worked, what didn't.

This is a partnership, not a replacement. Atlas is your second brain, not your boss.

What Atlas does

  1. Catalog Health. Drop in any catalog export. Atlas scores every ASIN, surfaces broken variants, missing images, weak content, and ranks fixes by dollar impact. Generates a fix file you can hand to whoever's editing listings.
  2. NIS Wizard. Drafts Amazon listings — title, bullets, description, backend keywords, A+ — to Novelle's voice. You review, edit, accept. Atlas learns from every edit so by month 3, the drafts need less editing.
  3. Marketing intelligence (coming soon). Drop in a PPC bulk file + search term report. Atlas joins them with catalog quality and tells you exactly where ad money is being wasted because of broken listings vs bad bids. Different problems need different fixes.

What Atlas does NOT do

  • Atlas does not push to Amazon. It generates files; you upload them. Your hand is always the last hand on the data.
  • Atlas does not change PPC bids automatically. It tells you what's wasted and what's underfunded; you make the call.
  • Atlas does not run unsupervised. Every output is reviewed before it ships.
  • Atlas does not replace you. It removes the mechanical parts of your job and gives you better tools for the parts that need judgment.
  • Atlas is not surveillance. It captures what gets decided and by whom — same as a Google Doc would. The point is institutional memory, not performance review.

How to use Atlas — step by step

First-time setup, then your daily/weekly flow.

1. First time you open Atlas

  1. Go to tlg-amazon-intelligence-dashboard.onrender.com in Chrome or Safari.
  2. You'll see a small chip in the top-right that says "Sign in". Click it.
  3. Enter your display name (your real name or the one you use professionally) and pick the role agency.
  4. The chip will now show your name. You're signed in for 90 days; no password needed.
  5. The brand dropdown will be locked to Novelle — that's correct. We're brand-scoped on purpose.

2. Drop your first file (Inputs tab)

  1. Click Inputs in the left sidebar.
  2. Drag any catalog/sales/PPC bulk/search term file onto the drop zone, or click Choose file.
  3. Atlas auto-detects the file type (catalog vs sales vs PPC bulk, etc.) and shows you what it found before the upload commits.
  4. Once uploaded, the file appears in the Ingestion history table below. The Staleness bar at the top of the page shows how old each file type is.
  5. For catalog files specifically, Atlas also runs the Catalog Health analyser automatically — see the toast notification.

3. Generate listings (NIS / Bulk Upload)

  1. Click Bulk Upload in the left sidebar (under Create).
  2. Upload a product template (Amazon NIS .xlsm) or use the existing one if you've already loaded it.
  3. Pick the styles you want drafted, click Generate.
  4. Review each field — title, bullets, description, backend keywords. You can Accept, Edit, or Reject.
  5. When you edit, Atlas asks the scope: just this listing, brand-wide, or category-wide. This is important — your choice trains the brand voice. Pick "brand-wide" only when you're sure the rule generalises.
  6. When done, click Submit session. Atlas prompts you for a short reflection (what was the biggest friction? what rule is worth codifying?). One sentence each is fine.
  7. Download the finished .xlsm and upload it to Seller Central as you would normally.

4. Check the audit trail (Memory tab)

  1. Click Memory in the left sidebar.
  2. Sessions tab: every session you've run, newest first. Click any row to see the full timeline of decisions for that sitting — what Atlas drafted, what you did with it, and the data snapshot at the moment.
  3. Decisions tab: a flat audit across all sessions. Filter by field name, ASIN, action (accept/edit/reject/comment), operator, or date range.
  4. Use this when a client asks "why did the title on B0XXXX change three weeks ago?" — Memory gives you the answer with the operator, the timestamp, and the data context.

5. Weekly cadence — suggested rhythm

  • Monday: PPC analysis. Pull the bulk file from Campaign Manager → Inputs → review the diagnosis → execute changes.
  • Tuesday: Listing updates in NIS. Generate drafts → edit to Novelle's voice → download for upload.
  • Wednesday: Catalog hygiene. Inventory export → Inputs → review the Catalog Health report → fix file out.
  • Friday: Weekly client report. Atlas drafts it; you review and edit.

6. Common questions

"I made a mistake on a session, can I undo?" The decision log can't be erased — that's the point of an audit trail. But you can log a corrective decision (a follow-up edit with a comment explaining the fix). The Memory tab shows both.

"How do I switch to a different operator on my team?" Click your name chip top-right → "Sign out" → next person signs in with their own display name. Same workstation is fine.

"What if Atlas drafts something wrong?" Reject it, leave a comment explaining why. That comment becomes a candidate brand rule. The more rejections, the faster Atlas learns.

"Can I see what gets sent to Amazon?" Atlas never sends anything to Amazon. You always download the file and upload it manually.

What gets captured

  • Every listing draft Atlas generates
  • Every edit you make to those drafts (and optionally why)
  • Every catalog issue surfaced and which ones got fixed vs deferred
  • Every fix file downloaded
  • The state of any ASIN at the moment a change was made (so we can compare before/after)
  • Which operator took which action and when

What does NOT get captured

  • Your screen
  • Your keystrokes
  • Anything outside Atlas
  • Anything personal

Support

  • Atlas itself / bugs / feature requests: Devang directly.
  • Amazon questions / strategy: still your area, not Atlas's.

What's coming next (4-6 weeks)

  • Photo memory — images persist across sessions, dedup automatically, link to ASINs.
  • Marketing diagnosis — full PPC × catalog cross-reference report with day-1 keyword setup logic, captured in the substrate.
  • Closed loop — see which decisions actually moved revenue / CVR / rank.

One ask before you start

When something is unclear or feels clunky, tell us in the moment, not in week 4. The product is in active development; the cost of fixing friction this month is roughly zero. The cost of fixing it three months from now is much higher.

Glad you're on this. Let's go.

یہ کیوں پڑھ رہے ہیں

آپ Novelle کی Amazon آپریشنز چلائیں گے — لسٹنگز، کیٹلاگ کی صفائی، PPC تجزیہ، ہفتہ وار رپورٹنگ۔ یہ کام آپ Helium 10، Excel، Seller Central یا اپنے لیپ ٹاپ کے notes ایپ میں کر سکتے ہیں۔ ہر دوسرا برانڈ ایسے ہی کرتا ہے۔

Novelle آپ سے یہی کام Atlas نامی پلیٹ فارم کے اندر کرنے کو کہہ رہا ہے جو ہم نے بنایا ہے۔ یہ دستاویز بتاتی ہے کہ Atlas کیا ہے، آپ کے لیے کیا بدلتا ہے، اور — کھل کر — آپ کو اس میں کیا ملتا ہے۔

مختصر یہ ہے:

  • آپ کے لیے: Atlas مشینی کام آپ کے کندھوں سے اتارتا ہے (ہفتہ وار رپورٹیں خودکار بنتی ہیں، کیٹلاگ مسائل ڈالر کے اثر کے حساب سے ترتیب میں آتے ہیں، لسٹنگز جلدی ڈرافٹ ہوتی ہیں)۔ آپ کا وقت فیصلوں پر زیادہ لگتا ہے، ڈیٹا جمع کرنے پر کم۔
  • Novelle کے لیے: آپ کا ہر فیصلہ محفوظ ہوتا ہے، تاکہ برانڈ کے پاس ایسی ادارتی یادداشت بنے جس پر آگے بنایا جا سکے۔

یہ شراکت ہے، نوکری کی جگہ پر کوئی متبادل نہیں۔ Atlas آپ کا دوسرا دماغ ہے، آپ کا باس نہیں۔

Atlas کیا کرتا ہے

  1. کیٹلاگ ہیلتھ۔ کوئی بھی کیٹلاگ ایکسپورٹ ڈالیں۔ Atlas ہر ASIN کو اسکور کرتا ہے، ٹوٹے ویریئنٹس، غائب تصاویر، کمزور مواد سامنے لاتا ہے، اور حلوں کو ڈالر کے اثر کے حساب سے ترتیب دیتا ہے۔ ایک fix file بناتا ہے جو آپ لسٹنگ ایڈیٹر کو دے دیں۔
  2. NIS Wizard۔ Amazon لسٹنگز ڈرافٹ کرتا ہے — ٹائٹل، بلٹس، تفصیل، بیک اینڈ کی ورڈز، A+ — Novelle کی آواز میں۔ آپ دیکھیں، ایڈٹ کریں، قبول کریں۔ Atlas ہر ایڈٹ سے سیکھتا ہے، اس لیے مہینہ 3 تک ڈرافٹس کو کم ایڈیٹنگ چاہیے ہوتی ہے۔
  3. مارکیٹنگ انٹیلیجنس (جلد آرہا ہے)۔ PPC bulk فائل + search term report ڈالیں۔ Atlas انہیں کیٹلاگ کوالٹی سے جوڑتا ہے اور بتاتا ہے کہ اشتہاری پیسہ کہاں ٹوٹی لسٹنگز کی وجہ سے ضائع ہو رہا ہے بمقابلہ غلط bids۔

Atlas کیا نہیں کرتا

  • Atlas Amazon پر کچھ نہیں بھیجتا۔ یہ فائلیں بناتا ہے؛ آپ اپ لوڈ کرتے ہیں۔ ڈیٹا پر آخری ہاتھ ہمیشہ آپ کا ہوتا ہے۔
  • Atlas PPC bids خود بخود نہیں بدلتا۔ یہ بتاتا ہے کہاں ضائع ہو رہا ہے اور کہاں کم ہے؛ فیصلہ آپ کا ہے۔
  • Atlas بغیر نگرانی نہیں چلتا۔ ہر آؤٹ پٹ بھیجنے سے پہلے ریویو ہوتا ہے۔
  • Atlas آپ کی جگہ نہیں لیتا۔ یہ آپ کے کام کے مشینی حصے کم کرتا ہے اور فیصلوں والے حصوں کے لیے بہتر اوزار دیتا ہے۔
  • Atlas نگرانی کا آلہ نہیں ہے۔ یہ صرف یہ ریکارڈ کرتا ہے کہ کیا فیصلہ ہوا اور کس نے کیا — جیسے کوئی Google Doc کرتا ہے۔ مقصد ادارتی یادداشت ہے، performance review نہیں۔

Atlas کیسے استعمال کریں — قدم بہ قدم

پہلی بار سیٹ اپ، پھر روزانہ/ہفتہ وار فلو۔

1. پہلی بار Atlas کھولنے پر

  1. Chrome یا Safari میں tlg-amazon-intelligence-dashboard.onrender.com پر جائیں۔
  2. اوپر دائیں کونے میں چھوٹا سا چپ نظر آئے گا جس پر "Sign in" لکھا ہوگا۔ اس پر کلک کریں۔
  3. اپنا ڈسپلے نام (اصل نام یا پیشہ ورانہ نام) درج کریں اور رول agency چنیں۔
  4. چپ پر اب آپ کا نام دکھے گا۔ 90 دن تک سائن ان رہیں گے؛ پاسورڈ نہیں چاہیے۔
  5. برانڈ ڈراپ ڈاؤن Novelle پر لاک ہوگا — یہی درست ہے۔ ہم جان بوجھ کر ایک برانڈ پر محدود ہیں۔

2. پہلی فائل ڈالیں (Inputs ٹیب)

  1. بائیں سائڈبار میں Inputs پر کلک کریں۔
  2. کوئی بھی catalog/sales/PPC bulk/search term فائل ڈراپ زون پر گھسیٹیں، یا Choose file پر کلک کریں۔
  3. Atlas فائل کی قسم خود پہچانتا ہے اور اپ لوڈ سے پہلے دکھاتا ہے کہ کیا ملا۔
  4. اپ لوڈ کے بعد فائل نیچے Ingestion history ٹیبل میں آجاتی ہے۔ صفحے کے اوپر Staleness bar بتاتا ہے ہر قسم کی فائل کتنی پرانی ہے۔
  5. کیٹلاگ فائلوں کے لیے Atlas خودکار طور پر Catalog Health analyser بھی چلاتا ہے۔

3. لسٹنگز بنائیں (NIS / Bulk Upload)

  1. بائیں سائڈبار میں Bulk Upload پر کلک کریں (Create کے نیچے)۔
  2. ایک پروڈکٹ ٹیمپلیٹ (Amazon NIS .xlsm) اپ لوڈ کریں یا موجودہ استعمال کریں۔
  3. جو styles ڈرافٹ کرنا چاہیں منتخب کریں، پھر Generate دبائیں۔
  4. ہر فیلڈ کو دیکھیں — ٹائٹل، بلٹس، تفصیل، بیک اینڈ کی ورڈز۔ آپ Accept، Edit، یا Reject کر سکتے ہیں۔
  5. جب آپ ایڈٹ کرتے ہیں، Atlas پوچھتا ہے کہ scope کیا ہے: صرف یہ لسٹنگ، پورے برانڈ پر، یا پوری کیٹگری پر۔ یہ اہم ہے — آپ کا انتخاب برانڈ کی آواز کو سکھاتا ہے۔ "برانڈ-وسیع" تب چنیں جب آپ کو یقین ہو کہ یہ اصول عام ہوگا۔
  6. کام مکمل ہونے پر Submit session پر کلک کریں۔ Atlas مختصر سوالات پوچھتا ہے (سب سے زیادہ رکاوٹ کیا تھی؟ کونسا اصول قابل ضبط ہے؟)۔ ہر کا ایک جملہ کافی ہے۔
  7. تیار .xlsm ڈاؤن لوڈ کر کے Seller Central پر معمول کے مطابق اپ لوڈ کریں۔

4. آڈٹ ٹریل دیکھیں (Memory ٹیب)

  1. بائیں سائڈبار میں Memory پر کلک کریں۔
  2. Sessions ٹیب: ہر سیشن جو آپ نے چلایا، نیا پہلے۔ کسی بھی قطار پر کلک کریں اور اس بیٹھک کی ساری ٹائم لائن دیکھیں — Atlas نے کیا ڈرافٹ کیا، آپ نے کیا کیا، اور اس لمحے کا ڈیٹا snapshot۔
  3. Decisions ٹیب: تمام سیشنز کا flat audit۔ فیلڈ نام، ASIN، action (accept/edit/reject/comment)، آپریٹر، یا تاریخ کے دائرے سے فلٹر کریں۔
  4. یہ تب استعمال کریں جب کلائنٹ پوچھے "تین ہفتے پہلے B0XXXX کا ٹائٹل کیوں بدلا؟" — Memory آپ کو آپریٹر، ٹائم اسٹیمپ، اور ڈیٹا سیاق کے ساتھ جواب دیتا ہے۔

5. ہفتہ وار شیڈول — تجویز کردہ تال

  • پیر: PPC تجزیہ۔ Campaign Manager سے bulk فائل → Inputs → diagnosis دیکھیں → تبدیلیاں کریں۔
  • منگل: NIS میں لسٹنگ اپ ڈیٹس۔ ڈرافٹس بنائیں → Novelle کی آواز میں ایڈٹ → ڈاؤن لوڈ → اپ لوڈ۔
  • بدھ: کیٹلاگ صفائی۔ Inventory ایکسپورٹ → Inputs → Catalog Health رپورٹ → fix file۔
  • جمعہ: ہفتہ وار کلائنٹ رپورٹ۔ Atlas ڈرافٹ بناتا ہے؛ آپ ریویو اور ایڈٹ کرتے ہیں۔

6. عام سوالات

"میں نے ایک سیشن میں غلطی کی، واپس لے سکتا ہوں؟" decision log کو مٹایا نہیں جا سکتا — یہی audit trail کا مقصد ہے۔ لیکن آپ ایک corrective decision لاگ کر سکتے ہیں (تبصرے کے ساتھ ایک follow-up edit)۔ Memory ٹیب دونوں دکھائے گا۔

"اپنی ٹیم میں دوسرے آپریٹر پر سوئچ کیسے کروں؟" اوپر دائیں اپنے نام کے چپ پر کلک کریں → "Sign out" → اگلا فرد اپنے ڈسپلے نام سے سائن ان کرے۔ ایک ہی workstation ٹھیک ہے۔

"اگر Atlas غلط ڈرافٹ کرے؟" Reject کریں، تبصرے میں وجہ لکھیں۔ یہ تبصرہ ممکنہ brand rule بن جاتا ہے۔ جتنے زیادہ rejects، Atlas اتنا تیز سیکھتا ہے۔

"کیا میں دیکھ سکتا ہوں جو Amazon کو بھیجا جاتا ہے؟" Atlas Amazon کو کبھی کچھ نہیں بھیجتا۔ آپ ہمیشہ فائل ڈاؤن لوڈ کر کے خود manually اپ لوڈ کرتے ہیں۔

  • ان drafts پر آپ کا ہر ایڈٹ (اور اختیاری طور پر وجہ)

  • ہر ڈاؤن لوڈ شدہ fix file
  • کسی بھی ASIN کی حالت جس لمحے تبدیلی ہوئی (تاکہ پہلے/بعد کا موازنہ ہو سکے)

Atlas — Novelle অপারেশনস অনবোর্ডিং

উদ্দিষ্ট: Novelle-এর Amazon অপারেশন চালাবে যে এজেন্সি টিম · প্রেরক: দেবাঙ্গ (Novelle) · তারিখ: মে ২০২৬

এটা কেন পড়ছেন

আপনি Novelle-এর Amazon অপারেশনস চালাবেন — লিস্টিং, ক্যাটালগ স্বাস্থ্য, PPC বিশ্লেষণ, সাপ্তাহিক রিপোর্টিং। এই কাজগুলো আপনি Helium 10-এ, Excel-এ, সরাসরি Seller Central-এ বা ল্যাপটপের notes অ্যাপে করতে পারেন। প্রায় প্রতিটি ব্র্যান্ড এভাবেই কাজ করে।

Novelle আপনাকে সেই একই কাজ Atlas নামের একটি প্ল্যাটফর্মের ভেতরে করতে বলছে, যেটি আমরা তৈরি করেছি। এই ডকুমেন্ট বুঝাবে Atlas কী, আপনার জন্য কী বদলায়, এবং — সরাসরি বলে — আপনার লাভ কী।

সংক্ষেপে:

  • আপনার জন্য: Atlas যান্ত্রিক কাজগুলো আপনার থেকে নিয়ে নেয় (সাপ্তাহিক রিপোর্ট স্বয়ংক্রিয়ভাবে তৈরি হয়, ক্যাটালগ সমস্যা ডলার-প্রভাব অনুযায়ী র‍্যাঙ্ক হয়, লিস্টিং দ্রুত খসড়া হয়)। আপনার সময় বেশি যাবে সিদ্ধান্তে, ডেটা একত্র করায় কম।
  • Novelle-এর জন্য: আপনার প্রতিটি সিদ্ধান্ত সংরক্ষিত হয় যাতে ব্র্যান্ডের একটি প্রাতিষ্ঠানিক স্মৃতি গড়ে ওঠে।

এটি অংশীদারিত্ব, প্রতিস্থাপন নয়। Atlas আপনার দ্বিতীয় মস্তিষ্ক, আপনার বস নয়।

Atlas কী করে

  1. Catalog Health. যেকোনো ক্যাটালগ এক্সপোর্ট ছেড়ে দিন। Atlas প্রতিটি ASIN-কে স্কোর করে, ভাঙা ভ্যারিয়েন্ট, অনুপস্থিত ছবি, দুর্বল কনটেন্ট সামনে আনে, এবং সমাধানগুলোকে ডলার-প্রভাব অনুযায়ী সাজায়। একটি fix file তৈরি করে যা লিস্টিং এডিটরের হাতে দেওয়া যায়।
  2. NIS Wizard. Amazon লিস্টিং খসড়া করে — টাইটেল, বুলেট, বিবরণ, ব্যাকএন্ড কীওয়ার্ড, A+ — Novelle-এর কণ্ঠে। আপনি দেখুন, এডিট করুন, accept করুন। প্রতিটি এডিট থেকে Atlas শেখে, তাই ৩ মাসের মধ্যে খসড়াগুলোতে কম এডিট লাগে।
  3. Marketing intelligence (শীঘ্রই আসছে). PPC bulk ফাইল + search term report দিন। Atlas এগুলোকে ক্যাটালগ মানের সাথে মিলিয়ে বলে কোথায় বিজ্ঞাপনের টাকা ভাঙা লিস্টিংয়ের জন্য নষ্ট হচ্ছে আর কোথায় ভুল bid-এর জন্য।

Atlas কী করে না

  • Atlas Amazon-এ কিছু পাঠায় না। এটি ফাইল তৈরি করে; আপনি আপলোড করেন। ডেটায় শেষ হাত সবসময় আপনার।
  • Atlas PPC bid স্বয়ংক্রিয়ভাবে বদলায় না। এটি বলে কোথায় অপচয় আর কোথায় কম খরচ; সিদ্ধান্ত আপনার।
  • Atlas নিরীক্ষণ ছাড়া চলে না। প্রতিটি আউটপুট পাঠানোর আগে পর্যালোচিত হয়।
  • Atlas আপনাকে প্রতিস্থাপন করে না। এটি আপনার কাজের যান্ত্রিক অংশ সরিয়ে দেয় এবং যেখানে বিচার দরকার সেখানে ভালো হাতিয়ার দেয়।
  • Atlas নজরদারি নয়। এটি শুধু রেকর্ড করে কী সিদ্ধান্ত হলো এবং কে নিলো — Google Doc যেমন করে। লক্ষ্য প্রাতিষ্ঠানিক স্মৃতি, performance review নয়।

Atlas কীভাবে ব্যবহার করবেন — ধাপে ধাপে

প্রথমবারের সেটআপ, তারপর প্রতিদিনের/সাপ্তাহিক ছন্দ।

১. প্রথমবার Atlas খোলার সময়

  1. Chrome বা Safari-তে tlg-amazon-intelligence-dashboard.onrender.com খুলুন।
  2. উপরের ডান কোণে একটি ছোট chip দেখবেন যেখানে লেখা "Sign in"। তাতে ক্লিক করুন।
  3. আপনার display name (আসল নাম বা পেশাগত নাম) দিন এবং role হিসেবে agency বেছে নিন।
  4. Chip-এ এখন আপনার নাম দেখাবে। ৯০ দিন sign-in থাকবেন; পাসওয়ার্ড লাগবে না।
  5. Brand dropdown Novelle-এ লক থাকবে — এটাই ঠিক। আমরা ইচ্ছাকৃতভাবে এক ব্র্যান্ডে সীমাবদ্ধ।

২. প্রথম ফাইল দিন (Inputs tab)

  1. বাম sidebar-এ Inputs ক্লিক করুন।
  2. যেকোনো catalog/sales/PPC bulk/search term ফাইল drop zone-এ টেনে আনুন, অথবা Choose file ক্লিক করুন।
  3. Atlas নিজেই ফাইলের ধরন শনাক্ত করে এবং আপলোডের আগে দেখায় কী পেয়েছে।
  4. আপলোডের পর ফাইলটি নিচে Ingestion history টেবিলে আসে। উপরে Staleness bar দেখায় প্রতিটি ধরনের ফাইল কত পুরোনো।
  5. ক্যাটালগ ফাইলের জন্য Atlas স্বয়ংক্রিয়ভাবে Catalog Health analyser-ও চালায়।

৩. লিস্টিং তৈরি করুন (NIS / Bulk Upload)

  1. বাম sidebar-এ Bulk Upload ক্লিক করুন (Create-এর নিচে)।
  2. একটি product template (Amazon NIS .xlsm) আপলোড করুন বা বিদ্যমানটি ব্যবহার করুন।
  3. যে styles খসড়া করতে চান বেছে নিন, তারপর Generate চাপুন।
  4. প্রতিটি field পর্যালোচনা করুন — title, bullets, description, backend keywords। আপনি Accept, Edit, বা Reject করতে পারেন।
  5. এডিট করলে Atlas scope জিজ্ঞেস করে: শুধু এই লিস্টিং, ব্র্যান্ড-জুড়ে, নাকি ক্যাটেগরি-জুড়ে। এটি গুরুত্বপূর্ণ — আপনার পছন্দ ব্র্যান্ড voice-কে শেখায়। "ব্র্যান্ড-জুড়ে" তখনই বেছে নিন যখন নিশ্চিত যে নিয়মটি সাধারণীকৃত।
  6. শেষ হলে Submit session চাপুন। Atlas সংক্ষিপ্ত প্রশ্ন করে (সবচেয়ে বড় বাধা কী ছিল? কোন নিয়ম codify-যোগ্য?)। প্রতিটিতে এক বাক্য যথেষ্ট।
  7. তৈরি .xlsm ডাউনলোড করে Seller Central-এ স্বাভাবিকভাবে আপলোড করুন।

৪. Audit trail দেখুন (Memory tab)

  1. বাম sidebar-এ Memory ক্লিক করুন।
  2. Sessions tab: আপনার চালানো প্রতিটি session, নতুন আগে। যেকোনো সারিতে ক্লিক করলে সেই বসার সিদ্ধান্তগুলোর পুরো timeline দেখা যাবে — Atlas কী খসড়া করেছে, আপনি কী করেছেন, এবং সেই মুহূর্তের data snapshot।
  3. Decisions tab: সব session-এর flat audit। Field name, ASIN, action (accept/edit/reject/comment), operator, বা তারিখের পরিসরে filter করুন।
  4. ক্লায়েন্ট যখন জিজ্ঞেস করে "তিন সপ্তাহ আগে B0XXXX-এর title কেন বদলেছিল?" — Memory আপনাকে operator, timestamp, এবং data context সহ উত্তর দেয়।

৫. সাপ্তাহিক ছন্দ — পরামর্শ

  • সোমবার: PPC বিশ্লেষণ। Campaign Manager থেকে bulk file → Inputs → diagnosis দেখুন → পরিবর্তন প্রয়োগ করুন।
  • মঙ্গলবার: NIS-এ listing update। Drafts তৈরি → Novelle-এর কণ্ঠে edit → ডাউনলোড → আপলোড।
  • বুধবার: Catalog hygiene। Inventory export → Inputs → Catalog Health report → fix file।
  • শুক্রবার: সাপ্তাহিক client report। Atlas খসড়া করে; আপনি পর্যালোচনা ও edit করেন।

৬. সাধারণ প্রশ্ন

"আমি একটি session-এ ভুল করেছি, undo করতে পারি?" Decision log মুছে ফেলা যায় না — audit trail-এর উদ্দেশ্যই এটি। কিন্তু আপনি একটি corrective decision log করতে পারেন (একটি follow-up edit মন্তব্য সহ)। Memory tab দুটোই দেখাবে।

"টিমে অন্য operator-এ switch করব কীভাবে?" উপরের ডানে আপনার name chip-এ ক্লিক → "Sign out" → পরের ব্যক্তি তার নিজের display name দিয়ে sign in করবে। একই workstation চলবে।

"Atlas ভুল খসড়া করলে?" Reject করুন, মন্তব্যে কারণ লিখুন। সেই মন্তব্য সম্ভাব্য brand rule হয়ে ওঠে। যত বেশি reject, Atlas তত দ্রুত শেখে।

"Amazon-এ কী পাঠানো হয় দেখতে পারি?" Atlas Amazon-এ কখনো কিছু পাঠায় না। আপনি সবসময় ফাইল ডাউনলোড করে নিজে manually আপলোড করেন।

কী সংরক্ষিত হয়

  • Atlas-এর তৈরি প্রতিটি listing draft
  • সেই draft-গুলোতে আপনার প্রতিটি edit (এবং ঐচ্ছিকভাবে কেন)
  • সামনে আসা প্রতিটি catalog issue এবং কোনগুলো ঠিক হলো বনাম স্থগিত
  • প্রতিটি ডাউনলোড হওয়া fix file
  • যেকোনো ASIN-এর অবস্থা যে মুহূর্তে পরিবর্তন হলো (আগে/পরে তুলনার জন্য)
  • কোন operator কখন কী action নিলো

কী সংরক্ষিত হয় না

  • আপনার screen
  • আপনার keystrokes
  • Atlas-এর বাইরের কিছু
  • ব্যক্তিগত কিছু

সহায়তা

  • Atlas / bugs / feature request: সরাসরি দেবাঙ্গকে।
  • Amazon প্রশ্ন / কৌশল: এটি এখনো আপনার ক্ষেত্র, Atlas-এর নয়।

সামনে কী আসছে (৪-৬ সপ্তাহ)

  • Photo memory — ছবি session-এর মধ্যে টিকে থাকে, স্বয়ংক্রিয় dedup, ASIN-এর সাথে link।
  • Marketing diagnosis — সম্পূর্ণ PPC × catalog cross-reference report, day-1 keyword setup logic সহ, substrate-এ সংরক্ষিত।
  • Closed loop — দেখুন কোন সিদ্ধান্ত প্রকৃতপক্ষে revenue / CVR / rank নাড়িয়েছে।

শুরুর আগে একটি অনুরোধ
