Atlas — Novelle Operations Onboarding

For: the agency team running Novelle's Amazon operations · From: Devang (Novelle) · Date: May 2026

Why you're reading this

You're going to run Novelle's Amazon operations — listings, catalog hygiene, PPC analysis, weekly reporting. You can do that work in Helium 10, in Excel, in Seller Central directly, and on your laptop's notes app. That's how every brand you've worked with does it.

Novelle is asking you to do most of that same work inside a platform we've built called Atlas. This doc explains what Atlas is, what changes for you, and — honestly — what's in it for you.

The short version:

  • For you: Atlas takes the grunt work off your plate (weekly reports auto-generate, catalog audits surface issues ranked by dollar impact, listings get drafted faster). You spend more time on judgment calls, less on assembling data.
  • For Novelle: every decision you make gets captured so the brand carries institutional memory you can build on. Future you, future agency hires, future Novelle hires — they can see what was tried, what worked, what didn't.

This is a partnership, not a replacement. Atlas is your second brain, not your boss.

What Atlas does

  1. Catalog Health. Drop in any catalog export. Atlas scores every ASIN, surfaces broken variants, missing images, weak content, and ranks fixes by dollar impact. Generates a fix file you can hand to whoever's editing listings.
  2. NIS Wizard. Drafts Amazon listings — title, bullets, description, backend keywords, A+ — to Novelle's voice. You review, edit, accept. Atlas learns from every edit so by month 3, the drafts need less editing.
  3. Marketing intelligence (coming soon). Drop in a PPC bulk file + search term report. Atlas joins them with catalog quality and tells you exactly where ad money is being wasted because of broken listings vs bad bids. Different problems need different fixes.

What Atlas does NOT do

  • Atlas does not push to Amazon. It generates files; you upload them. Your hand is always the last hand on the data.
  • Atlas does not change PPC bids automatically. It tells you what's wasted and what's underfunded; you make the call.
  • Atlas does not run unsupervised. Every output is reviewed before it ships.
  • Atlas does not replace you. It removes the mechanical parts of your job and gives you better tools for the parts that need judgment.
  • Atlas is not surveillance. It captures what gets decided and by whom — same as a Google Doc would. The point is institutional memory, not performance review.

How to use Atlas — step by step

First-time setup, then your daily/weekly flow.

1. First time you open Atlas

  1. Go to tlg-amazon-intelligence-dashboard.onrender.com in Chrome or Safari.
  2. You'll see a small chip in the top-right that says "Sign in". Click it.
  3. Enter your display name (your real name or the one you use professionally) and pick the role agency.
  4. The chip will now show your name. You're signed in for 90 days; no password needed.
  5. The brand dropdown will be locked to Novelle — that's correct. We're brand-scoped on purpose.

2. Drop your first file (Inputs tab)

  1. Click Inputs in the left sidebar.
  2. Drag any catalog/sales/PPC bulk/search term file onto the drop zone, or click Choose file.
  3. Atlas auto-detects the file type (catalog vs sales vs PPC bulk, etc.) and shows you what it found before the upload commits.
  4. Once uploaded, the file appears in the Ingestion history table below. The Staleness bar at the top of the page shows how old each file type is.
  5. For catalog files specifically, Atlas also runs the Catalog Health analyser automatically — see the toast notification.

3. Generate listings (NIS / Bulk Upload)

  1. Click Bulk Upload in the left sidebar (under Create).
  2. Upload a product template (Amazon NIS .xlsm) or use the existing one if you've already loaded it.
  3. Pick the styles you want drafted, click Generate.
  4. Review each field — title, bullets, description, backend keywords. You can Accept, Edit, or Reject.
  5. When you edit, Atlas asks the scope: just this listing, brand-wide, or category-wide. This is important — your choice trains the brand voice. Pick "brand-wide" only when you're sure the rule generalises.
  6. When done, click Submit session. Atlas prompts you for a short reflection (what was the biggest friction? what rule is worth codifying?). One sentence each is fine.
  7. Download the finished .xlsm and upload it to Seller Central as you would normally.

4. Check the audit trail (Memory tab)

  1. Click Memory in the left sidebar.
  2. Sessions tab: every session you've run, newest first. Click any row to see the full timeline of decisions for that sitting — what Atlas drafted, what you did with it, and the data snapshot at the moment.
  3. Decisions tab: a flat audit across all sessions. Filter by field name, ASIN, action (accept/edit/reject/comment), operator, or date range.
  4. Use this when a client asks "why did the title on B0XXXX change three weeks ago?" — Memory gives you the answer with the operator, the timestamp, and the data context.

5. Weekly cadence — suggested rhythm

  • Monday: PPC analysis. Pull the bulk file from Campaign Manager → Inputs → review the diagnosis → execute changes.
  • Tuesday: Listing updates in NIS. Generate drafts → edit to Novelle's voice → download for upload.
  • Wednesday: Catalog hygiene. Inventory export → Inputs → review the Catalog Health report → fix file out.
  • Friday: Weekly client report. Atlas drafts it; you review and edit.

6. Common questions

"I made a mistake on a session, can I undo?" The decision log can't be erased — that's the point of an audit trail. But you can log a corrective decision (a follow-up edit with a comment explaining the fix). The Memory tab shows both.

"How do I switch to a different operator on my team?" Click your name chip top-right → "Sign out" → next person signs in with their own display name. Same workstation is fine.

"What if Atlas drafts something wrong?" Reject it, leave a comment explaining why. That comment becomes a candidate brand rule. The more rejections, the faster Atlas learns.

"Can I see what gets sent to Amazon?" Atlas never sends anything to Amazon. You always download the file and upload it manually.

What gets captured

  • Every listing draft Atlas generates
  • Every edit you make to those drafts (and optionally why)
  • Every catalog issue surfaced and which ones got fixed vs deferred
  • Every fix file downloaded
  • The state of any ASIN at the moment a change was made (so we can compare before/after)
  • Which operator took which action and when

What does NOT get captured

  • Your screen
  • Your keystrokes
  • Anything outside Atlas
  • Anything personal

Support

  • Amazon questions / strategy: still your area, not Atlas's.
